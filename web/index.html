Como funciona

Três passos: emissão, registro e validação.

1

Emissão

A empresa emite o boleto no ERP e envia o código de barras para a API.

2

Blockchain

O código de barras é gravado na Stellar (Manage Data) em segundos.

3

Validação

Qualquer pessoa digita os 47 números e confere se o boleto é autêntico.

Por que o Boleto Guardian?

Menos fraude, mais transparência, sem complicação.

Sem dados técnicos

O usuário só precisa dos números do boleto. Nada de hash, Account ID ou chaves.

Imutável

Registro na blockchain Stellar: público, auditável e que não pode ser alterado.

ERP-agnóstico

Integra com qualquer ERP via API REST. Referência: TOTVS Protheus.
