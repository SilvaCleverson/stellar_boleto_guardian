Auditoria de Segurança — D-021

Relatório público da vulnerabilidade crítica identificada na Sprint 3 (Stellar 37 Degrees) e da correção implementada.

Projeto: Boleto Guardian · Correção: Sergio Artero (CTO) · Data: 20/05/2026

Relato detalhado — o que foi feito

Vulnerabilidade crítica identificada no fluxo administrativo do Boleto Guardian:

A ADMIN_API_KEY estava sendo armazenada no sessionStorage do navegador e enviada em texto claro no cabeçalho x-admin-key pela página registro.html. Isso permitia que qualquer pessoa com acesso ao DevTools visualizasse a credencial administrativa e potencialmente abusasse das APIs internas de registro/consulta on-chain.

Impacto

  • Exposição de credencial administrativa;
  • Possibilidade de chamadas indevidas a endpoints privilegiados;
  • Risco de registro indevido de boletos on-chain;
  • Risco crítico para integridade e controle operacional do sistema.

Correção aplicada

  • Remoção da ADMIN_API_KEY do frontend;
  • Remoção do uso de sessionStorage para segredo administrativo;
  • Eliminação do envio de x-admin-key pelo navegador;
  • ADMIN_API_KEY mantida exclusivamente no .env do backend;
  • Backend passa a absorver a autenticação administrativa internamente;
  • API isolada em rede Docker interna, evitando tráfego de segredo pelo cliente.

Referência: https://www.boletoguardian.xyz/auditoria-seguranca.html

Relatório consolidado: https://www.boletoguardian.xyz/auditoria-pentest.html

1. Achado crítico

Vulnerabilidade

Exposição da ADMIN_API_KEY (credencial administrativa secreta da API) no navegador, na página interna registro.html.

Como foi detectada

Revisão de segurança com DevTools do Chrome no fluxo de registro interno:

  • Application → Session Storage: chave gravada após "Salvar chave na sessão".
  • Network → POST /api/blockchain: cabeçalho x-admin-key com o segredo em texto claro.

Risco

AspectoImpacto
ConfidencialidadeCópia da chave admin e chamadas a endpoints privilegiados
IntegridadeRegistro indevido de boletos on-chain
SeveridadeCrítica

2. Correção realizada

Commit (correção final):

e7da273 — refactor: backend absorve ADMIN_API_KEY — nenhum header em trânsito

  • web/registro.html — removidos sessionStorage e x-admin-key no cliente.
  • backend/server.js — requireAdmin() usa apenas ADMIN_API_KEY no .env do servidor.
  • Docker — API na rede interna; segredo não trafega no navegador.

3. Resumo para o formulário

Vulnerabilidade: ADMIN_API_KEY + sessionStorage + x-admin-key (DevTools).

Risco: abuso de APIs administrativas (registro/consulta on-chain).

Correção: chave só no servidor; front sem segredo.

Commit: e7da273
