Security Audit — D-021

Public report on the critical vulnerability identified in Sprint 3 (Stellar 37 Degrees) and the implemented fix.

Project: Boleto Guardian · Fix: Sergio Artero (CTO) · Date: 20/05/2026

Detailed report — what was done

Critical vulnerability identified in the Boleto Guardian administrative flow:

The ADMIN_API_KEY was stored in browser sessionStorage and sent in plain text in the x-admin-key header from registro.html. Anyone with DevTools access could view the administrative credential and potentially abuse internal on-chain registration/query APIs.

Impact

  • Exposure of administrative credentials;
  • Possible unauthorized calls to privileged endpoints;
  • Risk of fraudulent on-chain slip registration;
  • Critical risk to system integrity and operational control.

Fix applied

  • Removed ADMIN_API_KEY from the front end;
  • Stopped using sessionStorage for administrative secrets;
  • Eliminated x-admin-key sent by the browser;
  • ADMIN_API_KEY kept only in backend .env;
  • Backend now handles admin authentication internally;
  • API isolated on internal Docker network so secrets never travel via the client.

Reference: https://www.boletoguardian.xyz/auditoria-seguranca-en.html

Consolidated report: https://www.boletoguardian.xyz/auditoria-pentest.html

1. Critical finding

Vulnerability

Exposure of ADMIN_API_KEY (secret administrative API credential) in the browser on the internal page registro.html.

How it was detected

Security review with Chrome DevTools during the internal registration flow:

  • Application → Session Storage: key saved after "Save key to session".
  • Network → POST /api/blockchain: x-admin-key header with the secret in plain text.

Risk

AspectImpact
ConfidentialityCopy admin key and call privileged endpoints
IntegrityUnauthorized on-chain slip registration
SeverityCritical

2. Fix implemented

Commit (final fix):

e7da273 — refactor: backend absorve ADMIN_API_KEY — nenhum header em trânsito

  • web/registro.html — removed sessionStorage and client x-admin-key.
  • backend/server.js — requireAdmin() uses ADMIN_API_KEY from server .env only.
  • Docker — API on internal network; secret never travels in the browser.
